# Communauté Synapse OMNIA

Bienvenue dans la communauté **Synapse OMNIA** — moteur de simulation 3D open source.

## Canaux de communication

| Canal | Usage | Lien |
|---|---|---|
| **GitHub Issues** | Bugs, demandes de fonctionnalités | [Issues](https://github.com/QuantumHacker10/Synapse/issues) |
| **GitHub Discussions** | Questions, idées, showcase | [Discussions](https://github.com/QuantumHacker10/Synapse/discussions) |
| **Pull Requests** | Contributions de code | [CONTRIBUTING.md](CONTRIBUTING.md) |
| **Discord** | Chat temps réel (à venir) | _Serveur en cours de création_ |

### Activer GitHub Discussions (mainteneurs)

Dans **Settings → General → Features**, activer **Discussions**. Catégories recommandées :

- **General** — questions générales
- **Ideas** — propositions de fonctionnalités
- **Show and tell** — projets et démos
- **Q&A** — questions techniques
- **Roadmap** — priorités et vision

## Contribuer

Consultez [CONTRIBUTING.md](CONTRIBUTING.md) pour le flux Git, les conventions de code et le versionnement.

En résumé :

1. Forker le dépôt
2. Créer une branche `feat/`, `fix/` ou `docs/` depuis `develop`
3. `dotnet build && dotnet test`
4. Ouvrir une PR vers `develop`

## Code de conduite

Ce projet adhère au [Contributor Covenant](CODE_OF_CONDUCT.md). En participant, vous acceptez de respecter ce code.

## Signaler un problème de sécurité

Ne pas ouvrir d'issue publique pour les vulnérabilités. Voir [SECURITY.md](SECURITY.md).

## Rejoindre l'équipe

Le projet accueille des contributeurs sur tous les domaines :

| Domaine | Compétences recherchées |
|---|---|
| Rendu Vulkan | G-DNN, L-DNN, shaders SPIR-V |
| Physique | Living laws, rigid bodies, CFD/FEA |
| IA | NEAT-G, ONNX, LLM integration |
| Studio UI | Avalonia, UX simulation |
| DevOps | CI/CD, multiplateforme, benchmarks |
| Documentation | Guides, tutoriels, traductions |

Ouvrez une Discussion **Introduction** pour vous présenter.
